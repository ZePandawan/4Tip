Documentation 4Tip Discord Bot

Un bot Discord complet et personnalisable pour la gestion de serveurs et les interactions communautaires.

Commandes Disponibles

/warn

Explication : Permet aux administrateurs de mettre un avertissement à un membre du serveur. L'avertissement peut être accompagné d'une raison.

Paramètres :

  • membre : Mentionnable (Membre à avertir) – Obligatoire
  • raison : String (Raison du warn) – Optionnel

/pokemon

Explication : Cette commande permet de capturer un Pokémon.

Paramètres :

  • Aucun paramètre requis.

/poke-inventory

Explication : Affiche l'inventaire du joueur contenant les Pokémon capturés.

Paramètres :

  • page : Number (Page de l'inventaire à afficher) – Optionnel

/add-streamer

Explication : Ajoute un streamer à la liste des streamers suivis.

Paramètres :

  • pseudo : String (Pseudo du streamer) – Obligatoire

/remove-streamer

Explication : Supprime un streamer de la liste des streamers suivis.

Paramètres :

  • pseudo : String (Pseudo du streamer) – Obligatoire

/list-streamer

Explication : Affiche la liste des streamers suivis.

Paramètres :

  • Aucun paramètre requis.

/define-channel

Explication : Permet de définir un salon spécifique pour les messages du bot (bienvenue, stream, etc.).

Paramètres :

  • type : String (Type de salon) – Obligatoire (Valeurs possibles : `welcome`, `stream`, `leave`, `admin`)
  • channel : Channel (Salon à définir) – Obligatoire

/lol-stats

Explication : Affiche les statistiques RANKED d'un joueur sur League of Legends.

Paramètres :

  • pseudo : String (Pseudo du joueur) – Obligatoire
  • tag : String (Tag du joueur) – Obligatoire
  • type : String (Type de classement) – Obligatoire (Valeurs possibles : `solo`, `flex`)

/lol-game-stats

Explication : Affiche les 5 derniers matchs d'un joueur sur League of Legends.

Paramètres :

  • pseudo : String (Pseudo du joueur) – Obligatoire
  • tag : String (Tag du joueur) – Obligatoire

/create-rules

Explication : Crée les règles du serveur et attribue un rôle aux membres acceptant ces règles.

Paramètres :

  • role : Role (Rôle à attribuer) – Obligatoire
